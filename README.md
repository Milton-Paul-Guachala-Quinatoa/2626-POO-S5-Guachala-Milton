## Descripción del sistema

Este proyecto consiste en un sistema básico de gestión de un restaurante desarrollado en Python utilizando Programación Orientada a Objetos.

El sistema permite registrar productos (platos y bebidas típicas ecuatorianas) y clientes, además de gestionar su información mediante una clase principal que organiza listas de objetos.

---

## Estructura del proyecto para mostrar en el repositorio en GitHub

├── restaurante_app/
│   ├── modelos/
│   │   ├── __init__.py
│   │   ├── producto.py
│   │   └── cliente.py
│   ├── servicios/
│   │   ├── __init__.py
│   │   └── restaurante.py
│   └── main.py
└── README.md

- modelos: contiene las clases Producto y Cliente
- servicios: contiene la clase Restaurante que gestiona las listas
- main.py: punto de inicio del programa

---

## Tipos de datos utilizados

- str: nombres, categorías, correos
- int: edad, cantidad de productos
- float: precios de productos
- bool: estado de disponibilidad o cliente activo
- list: almacenamiento de múltiples objetos

## Reflexión

El uso de identificadores descriptivos, tipos de datos adecuados y listas nos permite organizar mucho mejor los códigos que generamos, y gracias a eso también se nos facilita su lectura, y a la vez se puede mantener una estructura clara.

Por lo tanto, se puede decir que la programación modular es fundamental, ya que nos ayuda a separar el sistema en partes independientes, lo que mejora la comprensión, reutilización y corrección de los códigos en proyectos más grandes.
